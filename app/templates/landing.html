  1. 01

    Vault Sealed

    All credentials are encrypted at rest with AES-256-GCM. Without your master password, the data is mathematically indecipherable — even to us.

  2. 02

    Argon2 Key Derivation

    Your master password is processed through Argon2id (memory-hard, 64KB memory cost) entirely in the browser. It produces two keys: an AuthKey for login and a MasterEncryptionKey that never leaves your device.

  3. 03

    Server-Side Zero Trust

    The server receives only the AuthKey, then applies PBKDF2-HMAC-SHA256 with 200,000 iterations and a random 16-byte salt before storage. Even a full database breach yields zero usable credentials.

  4. 04

    Client-Side Decryption

    Vault records are decrypted in your browser using AES-256-GCM with the MasterEncryptionKey. The server only stores ciphertext blobs and IVs — it can never reconstruct your data.

Zero-knowledge encryption

The Web Crypto API handles AES-256-GCM encryption entirely in your browser. The server stores ciphertext only — no plaintext, no master password exposure. A database breach yields zero usable data.

Department sharing + audit

Share credentials within your department using derived symmetric keys. Every action is logged. Admins can track access patterns and revoke permissions instantly when teams change.

Adaptive TOTP 2FA

Even if an attacker extracts your keys, they cannot authenticate from another device without your physical TOTP token. Multi-factor authentication adds a critical defense-in-depth layer.

Architecture

Defense in depth, by design.

From memory-hard key derivation to strict transport security — every layer is deliberately hardened.

Argon2id + PBKDF2 layering

Client-side Argon2id (memory-hard) derives keys that the server then re-hashes with PBKDF2 (200K iterations). An attacker must defeat both to compromise a single account.

Strict transport layer

Nginx enforces TLSv1.2/1.3 only, with HSTS, X-Frame-Options DENY, and Cross-Origin Opener/Resource Policies (COOP/CORP) to isolate the browsing context and protect ephemeral keys.

Authenticated encryption (GCM)

AES-GCM provides authenticated encryption — any tampered ciphertext in the database fails the authentication tag check during decryption, immediately alerting the client to data integrity violations.
